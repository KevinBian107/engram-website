macOS · local vision model · incremental

Your handwritten notes,
as a knowledge graph.

Engram turns a Microsoft OneNote notebook — exported as one big PDF — into a local, Obsidian-native Markdown vault on your Mac. A local vision-language model reads every page (handwriting, typed text, diagrams, photos) and writes a faithful summary above the original page. A second pass links related notes — so your vault is a hierarchy that is also a graph.

🔒 Local-first · nothing leaves your Mac 🪶 The raw page is always the source of truth ♻️ Incremental & idempotent

From an opaque PDF to a navigable brain

You already write notes on an iPad with Apple Pencil in OneNote. You export a notebook to PDF (OneNote Windows → File → Export → Notebook → PDF). That gives you one giant, unsearchable file. Engram turns it into something you can actually think with.

A hierarchy that is also a graph

Notebook → Section → Note gives you the tree. A second, holistic linking pass reads every summary and connects related notes (shared concept · prerequisite · variation). The bundled macOS app renders the whole notebook as an interactive, force-directed graph — drag a node, or click one to light up its connections.

How a page becomes a note

Each note keeps its original page as a faithful per-note PDF leaf; the summary is a navigable projection written above it — always one scroll from the ink. Nothing is invented: typed text is preserved, handwriting is read, illegible parts are flagged.

What makes it different

Built for a personal knowledge base you actually trust — local, faithful, and reproducible.

🔒

Local & private

The vision model runs on-device via mlx-vlm (Qwen2.5-VL on Apple Silicon). No cloud, no upload, no API key — your notes never leave the Mac.

🪶

Faithful, not inventive

The raw page is the source of truth and is embedded in every note. Summaries reproduce what's on the page; illegible parts are flagged. A clean-reading but hallucinated summary is treated as a bug.

🕸️

Hierarchy and graph

Your OneNote structure becomes folders; a second linking pass adds ## Related cross-links — safe by construction (no dangling links, symmetric, capped) — so connections aren't only top-down.

♻️

Incremental & idempotent

Re-export and re-build: only changed or new notes are re-processed, keyed on a rendered-page fingerprint. An unchanged export is a true no-op — the model never even loads — and every run reports exactly what changed.

📝

Obsidian-native output

Plain Markdown on disk — frontmatter, [[wikilinks]], embedded PDFs, section/notebook overviews. Open the folder in Obsidian, or paste big slices into a long-context LLM (Karpathy's "second brain").

🖥️

Native macOS app

A SwiftUI app to import/replace a PDF (with a change report), browse notes by section, read each summary beside its original page — and explore the interactive knowledge graph.

Run it locally

Python 3.11+ and uv on macOS 14+ (Apple Silicon). The first run downloads the local model (~5 GB).

terminal
# 1 · set up the environment
uv sync --extra dev

# 2 · inspect the structure recovered from a OneNote PDF export
uv run python -m engram inspect "Skateboarding Notes.pdf"

# 3 · build (or incrementally update) the vault
uv run python -m engram build "Skateboarding Notes.pdf"

# 4 · add cross-note links → the knowledge graph
uv run python -m engram link "Skateboarding Notes"

uv run python -m engram status

Output lands in ~/Engram/<Notebook>/<Section>/<Title>.{md,pdf} — open ~/Engram in Obsidian.

Prefer a GUI? Build the native app:

macapp
cd macapp && ./build_app.sh && open Engram.app

Prototype status — the honest part

Engram's whole ethos is honesty over polish. This is a working v0.1 validated end-to-end on a real notebook (18 notes, 73 links) — but it's early, and several things are heuristic or not yet field-validated.

✓ Works today

  • OneNote → PDF export → split per-note PDF leaves
  • Local-VLM hierarchical summaries (LaTeX + code)
  • Obsidian vault: wikilinks, overviews, embedded PDFs
  • Holistic cross-link pass → the knowledge graph
  • Incremental rebuilds with a change report
  • CLI + a native macOS app

⚠ Prototype caveats

  • OneNote PDF export only — no other capture sources yet
  • macOS 14+ on Apple Silicon (~16 GB RAM for the 7B model)
  • Structure parsing is heuristic & locale-dependent (needs an English fallback)
  • Nested section subfolders are untested on a large notebook
  • Incremental determinism not yet field-validated on a real v2 export
  • One-way only (OneNote → vault); never written back
